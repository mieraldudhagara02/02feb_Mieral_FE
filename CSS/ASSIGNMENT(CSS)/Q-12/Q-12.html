QUESTION-12

How to maintain the CSS specifications?




-   If we want to apply more than one CSS rules to the same element, selector with the highest Specificity value will "win".
-   It's style declaration will be applied to that HTML element.

EXAMPLE:1

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Document</title>
<style>

    p{color: red; }

</style>
</head>
<body>

<p>THIS IS A PARAGRAPH TAG.</p>

</body>
</html>
-   In this Example we used "p" as selector and apply red color to the element.

RESULT:

THIS IS A PARAGRAPH TAG.


-   Now, we added class selector and specified a green color for this class.
-   Let's see what happen.

EXAMPLE:2

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Document</title>
<style>

    p{color: red; }
    .p1{color: green}

</style>
</head>
<body>

<p class="p1">THIS IS A PARAGRAPH TAG.</p>

</body>
</html>
-   In this Example we used ".p1" as selector and apply green color to the element.

RESULT:

THIS IS A PARAGRAPH TAG.


-   Now, Text is in Green color even though we have specified a red color for the element selector "p".
-   Because the class selector is given higher priority than "p" selector.
-   Now, we added id selector and specified a blue color for this class.

EXAMPLE:3

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Document</title>
<style>

    p{color: red; }
    .p1{color: green; }
    #P{color: blue; }

</style>
</head>
<body>

<p class="p1" id="P">THIS IS A PARAGRAPH TAG.</p>

</body>
</html>
-   In this Example we used "#p1" as selector and apply blue color to the element.

RESULT:

THIS IS A PARAGRAPH TAG.


-   Now, Text is in Blue color even though we have specified a green color for the element selector ".p1".
-   Because the id selector is given higher priority than ".p1" selector.

EXAMPLE:4

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Document</title>
<style>

    p{color: red; }
    .p1{color: green; }
    #P{color: blue; }

</style>
</head>
<body>

<p class="p1" id="P" style="color: blueviolet;">THIS IS A PARAGRAPH TAG.</p>

</body>
</html>
RESULT:

THIS IS A PARAGRAPH TAG.


-   The text is in blueviolet color, because the inline style is given the highest priority.
-   Thus, "Inline CSS" has higher specificity value than every selectors.

Specificity:
  1. Inline CSS
  2. ID selectors
  3. Class selectors
  4. Elements
How to Calculate Specificity?
- Start at 0, add 100 for each ID value.
- Add 10 for each class value (or pseudo-class or attribute selector).
- Add 1 for each element selector or pseudo-element.
- Inline style gets a specificity value of 1000, and is always given the highest priority!
